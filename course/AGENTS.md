# AGENTS 指南（`course`）

## 项目全貌
- 这是一个极简 Go 命令行程序，当前只有一个可执行入口：`main.go`。
- 模块定义在 `go.mod`：`module course`，Go 版本为 `1.24`。
- 代码没有分层目录（如 `cmd/`、`internal/`、`pkg/`），逻辑都在 `main()` 中串行执行。
- 当前无外部依赖，仅使用标准库 `fmt`；暂无服务边界、网络调用、数据库或配置系统。

## 关键代码路径
- 程序入口：`main.go` 的 `main()`。
- 执行流：
  1. 定义字符串 `s := "gopher"`
  2. 输出欢迎语 `fmt.Println("Hello and welcome, %s!", s)`
  3. `for i := 1; i <= 5; i++` 循环输出 `100/i`
- 现状细节：欢迎语使用了 `Println` + `%s`，会把 `%s` 当普通文本输出，而不是格式化占位符。

## 开发与验证工作流
- 运行程序（已验证可执行）：
  ```bash
  cd /Users/panzd/course/course
  go run .
  ```
- 全量测试/构建检查（当前会失败）：
  ```bash
  cd /Users/panzd/course/course
  go test ./...
  ```
- 当前失败点（来自 `go test`）：`main.go:13` 提示 `fmt.Println` 可能误用格式化指令 `%s`。
- 若要修复该告警，通常改为 `fmt.Printf("Hello and welcome, %s!\n", s)` 或 `fmt.Println("Hello and welcome,", s)`。

## 本项目约定（从现有代码可观察）
- 单文件原型风格：优先直接在 `main.go` 修改，再考虑抽函数。
- 注释中包含 GoLand 模板提示（`//TIP ...`）；这类注释不参与运行逻辑。
- 当前输出是演示性质，未看到日志框架、错误包装、配置注入等工程化约定。

## 集成点与扩展注意事项
- 外部集成点：当前没有（无 HTTP、RPC、DB、消息队列、文件 I/O）。
- 新增第三方包时需同步更新 `go.mod`/`go.sum`，并确保 `go test ./...` 通过。
- 若开始扩展为多文件结构，建议先保持入口 `main.go` 精简，只做参数解析与流程编排。
